# Доска Объявлений (Bulletin Board Service)

Сервис для курса "Разработка Безопасного Программного Обеспечения"

## Описание

Сервис доски объявлений, позволяющий пользователям создавать объявления, общаться через сообщения и оставлять жалобы на объявления.

## Технологии

- Java 17
- Spring Boot 3.5.5
- PostgreSQL
- Maven

## Основные сущности

- **Listing** - объявление, принадлежит пользователю и категории
- **Category** - категория объявления
- **User** - пользователь системы
- **Message** - сообщение в переписке между пользователями по объявлению
- **Report** - жалоба на объявление (один пользователь не может жаловаться дважды на одно объявление)

## API Endpoints

### Listing
- `GET /api/listings` - получить все объявления
- `GET /api/listings/{id}` - получить объявление по ID
- `POST /api/listings` - создать объявление
- `PUT /api/listings/{id}` - обновить объявление
- `DELETE /api/listings/{id}` - удалить объявление

### Category
- `GET /api/categories` - получить все категории
- `GET /api/categories/{id}` - получить категорию по ID
- `POST /api/categories` - создать категорию
- `PUT /api/categories/{id}` - обновить категорию
- `DELETE /api/categories/{id}` - удалить категорию

### User
- `GET /api/users` - получить всех пользователей
- `GET /api/users/{id}` - получить пользователя по ID
- `POST /api/users` - создать пользователя
- `PUT /api/users/{id}` - обновить пользователя
- `DELETE /api/users/{id}` - удалить пользователя

### Message
- `GET /api/messages` - получить все сообщения
- `GET /api/messages/{id}` - получить сообщение по ID
- `POST /api/messages` - создать сообщение
- `PUT /api/messages/{id}` - обновить сообщение
- `DELETE /api/messages/{id}` - удалить сообщение

### Report
- `GET /api/reports` - получить все жалобы
- `GET /api/reports/{id}` - получить жалобу по ID
- `POST /api/reports` - создать жалобу
- `PUT /api/reports/{id}` - обновить жалобу
- `DELETE /api/reports/{id}` - удалить жалобу

### Бизнес-операции
- `GET /api/business/listings/category/{categoryId}` - получить объявления по категории
- `GET /api/business/listings/user/{userId}` - получить объявления пользователя
- `GET /api/business/listings/price-range?minPrice={min}&maxPrice={max}` - получить объявления по диапазону цен
- `GET /api/business/listing/{listingId}/statistics` - получить статистику объявления (количество сообщений и жалоб)
- `GET /api/business/listing/{listingId}/conversation` - получить переписку по объявлению

## Настройка

1. Убедитесь, что PostgreSQL установлен и запущен
2. Создайте базу данных: `CREATE DATABASE bulletin_board;`
3. Настройте переменные окружения:
   - `DB_USERNAME` - имя пользователя PostgreSQL (по умолчанию: postgres)
   - `DB_PASSWORD` - пароль PostgreSQL (по умолчанию: postgres)
4. Запустите приложение: `mvn spring-boot:run`

## Безопасность

### Регистрация пользователя
Для регистрации используйте endpoint `/api/auth/register`. Пароль должен соответствовать требованиям:
- Минимум 8 символов
- Хотя бы одна заглавная буква
- Хотя бы одна строчная буква
- Хотя бы одна цифра
- Хотя бы один специальный символ (!@#$%^&*(),.?":{}|<>)

### Аутентификация
Все API endpoints (кроме `/api/auth/register` и `/api/auth/login`) требуют Basic Authentication:
- Username: ваш username
- Password: ваш пароль

### Роли
- `USER` - обычный пользователь (по умолчанию)
- `ADMIN` - администратор

Все endpoints доступны для ролей USER и ADMIN.
