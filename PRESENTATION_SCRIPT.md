## Возможные вопросы и ответы

**Q: Как работает механизм refresh токенов?**
**A:** "При обновлении токенов система проверяет активность сессии в БД. Если сессия активна, старая помечается как REVOKED, создается новая сессия с новым refresh токеном. Старый refresh токен больше не может быть использован, что защищает от replay атак."

**Q: Почему access token живет 15 минут?**
**A:** "Короткий срок жизни access token снижает риск при компрометации. Если токен украден, он будет недействителен через 15 минут. Для обновления используется долгоживущий refresh token."

**Q: Как проверить цепочку сертификатов?**
**A:** "Можно использовать команду `openssl verify` или открыть сертификат в браузере и посмотреть путь сертификации. Должны быть видны все три звена: Server → Intermediate CA → Root CA."

**Q: Где хранятся секреты в CI/CD?**
**A:** "В GitHub Secrets или GitLab Variables. Keystore кодируется в base64 и хранится как секрет, пароль также хранится как секрет. Они никогда не попадают в код или логи."

**Q: Что происходит при истечении refresh token?**
**A:** "При истечении refresh token сессия автоматически помечается как EXPIRED. Пользователю нужно будет заново выполнить вход через `/auth/login`."
